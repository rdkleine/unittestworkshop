Wat is een Unit Test

Unit of work (function / class / component)
Geisoleerd testen (mocking / DI / Test Db / Sandbox)
Assertion of het resultaat of gedrag overeenkomt met de verwachting.

Waarom Unit Testen

Vroegtijdig in het ontwikkelstadion bugs vinden

  • Vangnet wanneer wijzigingen in de codebase worden gemaakt. Voorkomen van nieuw geintroceerde bugs.
  • Onderhoudbaarheid van de codebase vergroten
  • Gedrag kunnen valideren in foutsituaties
  • Bij elke PR de werking van de code kunnen valideren
    • Hoe schrijven we Unit Tests
    • Identificeer een unit of code die getest moet worden
    • Schrijf een test methode die de functionaliteit uitvoerd
    • Gebruik Assertion om te valideren of het verwachte resultaat of gedrag toont
    • Testen van edge cases en exceptions
    Voorbeeld van een Unit Test 
    // Test Method
public void TestAddition()
{
    // Arrange
    Calculator calculator = new Calculator();
    
    // Act
    int result = calculator.Add(2, 3);
    
    // Assert
    Assert.AreEqual(5, result);
}
// Code under test
public class Calculator
{
    public int Add(int a, int b)
    {
        return a + b;
    }
}
    Best prectices
    • Schrijf testen die onafhankelijk zijn van elkaar
    • Test voor resultaat/gedrag niet de implementatie details
    • Houdt testen klein en gefocussed
    • Gebruik een naming convention en gebruik deze consistent
    Hulpmiddelen
    • Code coverage (in editor)
